# ffhoreca — Tips from trips

Личная интерактивная карта путешествий: страны, города, места (отели, еда, бары, достопримечательности) и маршруты между городами.

---

## Что видит пользователь

1. **Карта мира** (MapLibre + тайлы CARTO/OSM).
2. **Цвет стран** — посещённые страны подсвечены зелёным; Азербайджан и Катар — жёлтым (только пересадка).
3. **Города** — точка на карте; при приближении — оранжевая заливка границы (если есть файл `public/geo/cities/{cityId}.json`).
4. **Места** — цветные маркеры по категории; при клике — карточка с фото, адресом, оценкой Google, текстом.
5. **Табы** — фильтр: всё, Места, Города, Жильё, Еда, Бары, Аэропорты.
6. **Маршруты** — линии между городами и анимированная иконка транспорта (самолёт, поезд, автобус, лодка).
7. **Поиск** — строка над картой для перехода к городу или месту.

В **режиме админа** (`?token=…` в URL) доступны: добавление места и маршрута, список «Управление», редактирование и удаление.

---

## Откуда берутся данные

| Данные | Источник |
|--------|----------|
| Каталог городов и мест | API `GET /api/catalog` (SQLite на сервере) |
| Локальный fallback | `localStorage` (если API не настроен) |
| Границы городов | Статические GeoJSON в `public/geo/cities/` |
| Границы стран | `world-atlas` (countries-10m), заливка на клиенте |
| Поиск адреса в форме «Новое место» | Google Places API (если есть ключ) или Photon (OSM) |
| Фото мест | Загрузка на сервер `POST /api/photos` → URL `/uploads/…` на Fly volume `/data/uploads` |

Хардкодный каталог в `src/data/catalog.ts` пустой — всё живёт на бэкенде.

При добавлении места город может **создаваться автоматически** из ответа Google Places (если его ещё нет в базе) и сохраняться вместе с местом (`POST /api/places` с полем `city`).

## Архитектура

```
Браузер (Vite + React)
  ├── WorldMap.tsx      — карта, слои, маркеры
  ├── App.tsx           — каталог, фильтры, модалки
  ├── AddPlaceModal     — новое место (поиск → поля)
  ├── PlaceModal        — просмотр / редактирование места
  ├── AddRouteModal     — новый маршрут
  ├── ManagerModal      — списки маршрутов и мест
  └── UserRoutes.tsx    — линии и анимация маршрутов

Сервер (Hono + better-sqlite3), Fly.io
  ├── /api/catalog
  ├── /api/places
  ├── /api/routes
  ├── /api/photos
  └── /uploads/:filename

Постоянный том Fly: /data
  ├── catalog.sqlite
  └── uploads/
```

Фронтенд обычно на **GitHub Pages**, бэкенд на **Fly.io**. Связь через `VITE_API_BASE_URL` и `VITE_ADMIN_TOKEN` (см. `.env.example`).

## Ключевые сценарии

### Добавить место

1. Админ открывает «+ Место».
2. Вводит название в поле поиска → выбирает подсказку (Google/Photon).
3. Подставляются адрес, координаты, оценка, город (в т.ч. новый, если его не было в каталоге).
4. Заполняет категории, краткое описание, впечатления, при желании — фото.
5. `POST /api/places` (+ `city` при необходимости) → обновление каталога на карте.

### Добавить маршрут

1. «+ Маршрут» → выбор городов (с поиском), тип транспорта.
2. `POST /api/routes` → `GET /api/routes` → отрисовка на карте.

### Таб «Места»

Показываются только места категории `attraction`, но **города с такими местами** должны оставаться на карте: метка, заливка границы (если есть GeoJSON), плюс маркеры мест.

## Деплой бэкенда

```bash
fly deploy
```

В `Dockerfile`: `DATABASE_PATH=/data/catalog.sqlite`, `UPLOADS_DIR=/data/uploads`.  
Том `ffhoreca_data` смонтирован в `/data` (`fly.toml`).

Подробнее: `docs/BACKEND-DEPLOY.md`, `docs/BACKEND-DEPLOY-CLOUD.md`.
